03 / 12 Baseline  ·  gt today, transcribed faithfully gt · design refinement

The current overview baseline

Reading what's actually there. The screen is already mutt-disciplined — dense, scannable, monochrome with three accent slots. The work is sharpening, not redrawing.

~/git/gt/gt — gt
On branch main Untracked (2) ? .gitignore ? go.mod Unstaged (1) M main.go Staged (1) M README.md Recent commits cfdc578 (HEAD → main) Refactor data pipeline 4bc871d Update dependencies a978566 Fix authentication bug 8ecd59d Add CI configuration c3a0f55 Initial commit c=commit d=diff s=stage u=unstage t=tag ?=help q=quit

What's already right

  • Section headers bold & warm, file rows neutral — correct hierarchy
  • Section counts in parens — mutt pattern, instantly readable
  • X/Y indicators (?/M/A/D) carry meaning by glyph, not just color
  • Context-aware hint bar at bottom — teaches the keymap

What I want to tighten

  • All four section headers are the same amber — the eye can't tell “has work” from “empty”
  • Cursor row spans the full terminal width even when content is short — reads as a bar, not a pin
  • No visible relationship between sections; they float in the void
  • HEAD ref is amber-bold which fights the section headers for attention
  • Indicator column has no internal alignment between sections
Goal: same density, sharper hierarchy, accents that earn their use. Don't make it pretty — make it correct.

05 / 12 Overview · refined  ·  the primary proposal gt · design refinement

Overview, refined primary

Same density, six surgical changes. A continuous left rail anchors the screen; section headers earn their amber by signaling work-to-do; the cursor pins instead of bars; HEAD steps back so branch can step forward.

~/git/gt/gt — gt
┊ main origin/main ↑2 ↓0 v0.1.0+a3f9 ┊ Untracked · 2 ┊ ? .gitignore ┊ ? go.mod ┊ Unstaged · 1 +12/-3 ┊ M main.go +12/-3 ┊ Staged · 1 +24/-8 ┊ M README.md +24/-8 ┊ Recent commits ┊ ▶ cfdc578 3m Refactor data pipeline (main, origin/main) ┊ ▶ 4bc871d 30m Update dependencies ┊ ▶ a978566 2h Fix authentication bug s=stage t=tag c=commit d=diff ?=help q=quit

Six refinements

  • Left rail. A 1-cell ┊ column ties the screen together. Bright on sections with work, dimmed when empty.
  • Header rhythm. Counts as · 2 after the name — lighter than parens, mutt-style.
  • Diff-stat on headers. +12 / -3 next to the section name — you see the size of the change before drilling in.
  • Right-aligned per-file stats. Uses the empty space we already have.
  • "Recent commits" demoted. Soft fg, not amber — it's reference, not action.
  • Commit row layout. ▶ sha 3m title (refs) — age in 4-cell field, refs trail.

What stayed

  • Every keystroke. The model.go contract is unchanged.
  • Section order. Untracked → Unstaged → Staged → Working tree → Log.
  • X/Y indicators. ? M A D still carry the same colors.
The rail is doing real work: it's the section attendance light. Bright = there are files here. Ghost = empty section, still navigable. You can read the screen at a glance from across the room.
Cursor pins to content width — doesn't bleed across the terminal. diff vs baseline: 6 changes, 0 new keys

07 / 12 Inline diff  ·  unified, in-process gt · design refinement

Inline diff — unified new

Press d, the diff opens in gt instead of shelling to less. Same cell grid, same keys, navigation by hunk. q returns. Falls back to less for very long diffs (configurable).

gt — diff main.go (unstaged)
┊ diff main.go · M unstaged +12/-3 hunk 2/3 ┊ ┊ @@ -42,7 +42,12 @@ func (m *Model) refresh() error { ┊ 42 if err := m.loadStatus(); err != nil { ┊ 43 return fmt.Errorf("status: %w", err) ┊ 44 } ┊ 45 - m.log = m.loadLog(50) ┊ 46 - return nil ┊ 47 - } ┊ + log, err := m.loadLog(50) ┊ + if err != nil { ┊ + return fmt.Errorf("log: %w", err) ┊ + } ┊ + m.log = log ┊ + m.lastRefresh = time.Now() ┊ + return nil ┊ + } ┊ 48 ┊ 49 // loadStatus runs git status --porcelain=v2 and parses. ┊ ┊ @@ -78,3 +83,8 @@ type Status struct { ┊ 78 Untracked []FileEntry ┊ 79 Unstaged []FileEntry ┊ 80 Staged []FileEntry ┊ + Branch string ┊ + Upstream string ┊ ───────────────────────────────────────────────────────────────────────────────────────────────────── j/k=line ]/[=hunk s=stage·hunk v=split e=editor L=less q=back

Anatomy

  • Header line identifies file + section + diff stat + hunk position. Same rail as overview.
  • Old/new line numbers as two 4-cell columns — stays aligned even with 4-digit lines.
  • Sign column (+/-/ ) is the one piece carrying color.
  • Soft tinted background on changed rows — 10% alpha; cursor row goes to 18%. Color isn't doing the heavy lifting.
  • Hunk separator is the existing rail glyph at fg-ghost luminance — structural, near-invisible.

New keys (small additions)

  • ] / [ jump to next/prev hunk
  • s here stages just this hunk — the long-deferred git add -p entry point
  • v toggles split (next slide)
  • L hands off to $PAGER if you want the full less experience
The diff doesn't replace less; it makes the most common diff (one file, a few hunks) feel native. Long diffs still belong in the pager.
Implementation: parse git diff --no-color -U3 output. ~150 lines. fallback threshold: configurable (default 1000 lines)
08 / 12 Inline diff  ·  split — same data, parallel gt · design refinement

Inline diff — split v toggles

Same hunk, two columns. Old left, new right. The vertical box-drawing rule between them is the only chrome. Press v in unified to switch — setting persists per-session.

gt — diff main.go · split
┊ diff main.go · M unstaged · split +12/-3 hunk 2/3 ┊ ┊ @@ -42,7 +42,12 @@ func (m *Model) refresh() error { ┊ 42 if err := m.loadStatus(); err != nil { │ 42 if err := m.loadStatus(); err != nil { ┊ 43 return fmt.Errorf("status: %w", err) │ 43 return fmt.Errorf("status: %w", err) ┊ 44 } │ 44 } ┊ 45 m.log = m.loadLog(50) │ 45 log, err := m.loadLog(50) ┊ 46 return nil │ 46 if err != nil { ┊ 47 } │ 47 return fmt.Errorf("log: %w", err) ┊ │ 48 } ┊ │ 49 m.log = log ┊ │ 50 m.lastRefresh = time.Now() ┊ │ 51 return nil ┊ │ 52 } ┊ 48 │ 53 ┊ 49 // loadStatus runs git status --porcelain=v2. │ 54 // loadStatus runs git status --porcelain=v2. ┊ ┊ @@ -78,3 +83,8 @@ type Status struct { ┊ 78 Untracked []FileEntry │ 83 Untracked []FileEntry ┊ 79 Unstaged []FileEntry │ 84 Unstaged []FileEntry ┊ 80 Staged []FileEntry │ 85 Staged []FileEntry ┊ │ 86 Branch string ┊ │ 87 Upstream string ┊ ───────────────────────────────────────────────────────────────────────────────────────────────────────────────── j/k=line ]/[=hunk s=stage·hunk v=unified L=less q=back

When to prefer split

  • Refactors where lines move — alignment is much faster to read
  • Wide enough terminal (≥120 cols); auto-falls-back to unified below 100
  • You're in a code-review mood, not a stage-and-go mood

Implementation note

  • Server-side line pairing: walk the unified diff, assign each -/+ pair the same row index where possible; bare - or + get an empty cell on the other side.
  • The vertical │ rail is at fg-ghost luminance; it disappears when you stop looking at it — structural, not decorative.
  • Cursor highlight applies to both columns of the same logical row.
The split is one pixel of CSS apart from the unified. Same model, same parser, same keys. v toggles. That's the whole feature.
Both modes route hunk-stage through the same plumbing. ≥120 cols recommended for split
09 / 12 Stash  ·  a fourth section, two new keys gt · design refinement

Stash, mutt-style new feature

Stash isn't a separate screen — it's a fourth section, sitting between Staged and Working tree. Empty by default; appears when you have stashes. Cursor moves through them like any other row.

~/git/gt/gt — gt
┊ main origin/main ↑2 ↓0 ┊ ┊ Untracked · 2 ┊ ? .gitignore ┊ ? go.mod ┊ ┊ Unstaged · 1 +12/-3 ┊ M main.go +12/-3 ┊ ┊ Staged · 1 +24/-8 ┊ M README.md +24/-8 ┊ ┊ Stash · 3 ┊ ▶ stash@{0} 3h WIP on main: tweak commit prompt ┊ ▶ stash@{1} 2d debug logging in update.go ┊ ▶ stash@{2} 6d experiment: hunk-level staging ┊ ┊ Recent commits ┊ ▶ cfdc578 3m Refactor data pipeline (main) ┊ ▶ 4bc871d 30m Update dependencies ┊ ▶ a978566 2h Fix authentication bug ┊ ┊ ──────────────────────────────────────────────────────────────────────────────────────────────── d=show p=pop a=apply D=drop l=expand S=stash·new ?=help q=quit

Why a section, not a screen

  • Stash is just state, like Untracked and Staged. It belongs in the inventory.
  • You see it without asking. The first time you stash you'll find it without reading docs.
  • Empty stash = section hidden — same rule as “Staged · empty”.

The grammar holds

  • l on a stash row expands its file list (just like a commit)
  • d diffs the stash (just like a commit)
  • t tags it — ;d diffs all tagged stashes back-to-back
  • Mauve ┊ rail visually separates “parked work” from active changes

Two new keys

  • S — stash all. Opens a one-line message prompt, same UX as c. Empty message = no message.
  • p — pop. Only valid on a stash row. Asks y/n if stash isn't on current branch.
  • a apply, D drop — only on stash rows; out of the way otherwise.
Mauve rail = the same color as the tag glyph — reinforces “set aside” semantics. git stash list / show / pop / apply / drop / push -m

Cursor & rail variants explore

Three options for the cursor row and the section attendance signal. My recommendation is the middle one (full-row dim fill + bright rail). The others are valid — pick what feels right at your terminal.

A · gutter caret
gt — A
┊ main origin/main ↑2 ↓0 ┊ ┊ Untracked · 2 ┊ ? .gitignore ┊ ? go.mod ┊ ┊ Unstaged · 1 ▎┊ M main.go ┊ ┊ Staged · 1 ┊ M README.md ┊ ┊ Recent commits ┊ ▶ cfdc578 3m Refactor ┊ ▶ 4bc871d 30m Update deps

A ▎ caret in column 0 marks the cursor. No fill. Lightest visual weight. Best for very dense sessions; cursor can get lost on long lines.

B · row fill (recommended)
gt — B
┊ main origin/main ↑2 ↓0 ┊ ┊ Untracked · 2 ┊ ? .gitignore ┊ ? go.mod ┊ ┊ Unstaged · 1 ┊ M main.go ┊ ┊ Staged · 1 ┊ M README.md ┊ ┊ Recent commits ┊ ▶ cfdc578 3m Refactor ┊ ▶ 4bc871d 30m Update deps

Soft cool-grey row fill (#2a2a35). Rail still bright. Cursor is found instantly even on busy screens. The default.

C · inverted
gt — C
┊ main origin/main ↑2 ↓0 ┊ ┊ Untracked · 2 ┊ ? .gitignore ┊ ? go.mod ┊ ┊ Unstaged · 1 ┊ M main.go ┊ ┊ Staged · 1 ┊ M README.md ┊ ┊ Recent commits ┊ ▶ cfdc578 3m Refactor ┊ ▶ 4bc871d 30m Update deps

Reverse video on the cursor row. Maximum punch. Reads as “selected” in mutt & ranger; some find it loud. Available as a tweak.

All three render correctly on truecolor terminals; B and C degrade to ANSI 241 / reverse on 16-color. controlled by cursor.style tweak (gutter · row · invert)
